# Contributing to Awesome Modern Data Stack

Thank you for your interest in contributing to this curated list! Your contributions help the data community discover and learn about the best tools in the modern data stack ecosystem.

## How to Contribute

### Adding a New Tool

1. **Fork this repository** and create a new branch for your contribution.

2. **Check if the tool already exists** in the list before adding it.

3. **Add the tool in the appropriate category** following the existing format:
   ```markdown
   - [Tool Name](https://tool-url.com/) - Brief description of what the tool does.
   ```

4. **Follow these guidelines:**
   - Descriptions should be concise (one sentence, max ~15 words)
   - Start descriptions with a capital letter and end with a period
   - Descriptions should be objective, not promotional
   - Place the tool alphabetically within its category (or group by open-source/commercial if the category is structured that way)
   - Ensure the URL is the official website, not a third-party link

5. **Submit a Pull Request** with a clear title and description.

### Criteria for Inclusion

Tools should meet the following criteria to be included:

- **Relevance**: The tool must be relevant to the modern data stack ecosystem
- **Active Development**: The tool should be actively maintained (commits within the last 6 months for open-source projects)
- **Documentation**: The tool should have reasonable documentation
- **Accessibility**: The tool should be generally available (not in private beta without a waitlist)
- **Reputation**: The tool should have some level of adoption or recognition in the data community

### What NOT to Include

- Abandoned or deprecated projects
- Tools that are purely promotional without real functionality
- Duplicate tools that don't offer meaningful differentiation
- Personal projects without community adoption
- Tools that violate ethical guidelines or are used primarily for malicious purposes

## Updating Existing Entries

If you notice an entry needs updating (broken link, outdated description, company acquisition, etc.), please submit a PR with the correction.

## Suggesting New Categories

If you think a new category should be added:

1. Open an issue first to discuss the proposed category
2. Explain why the category is needed and what tools would go in it
3. Once approved, submit a PR adding the category with at least 3 tools

## Reporting Issues

If you find:
- Broken links
- Incorrect information
- Deprecated tools
- Duplicate entries

Please open an issue or submit a PR with the fix.

## Pull Request Guidelines

1. **One change per PR**: Keep PRs focused on a single addition or fix
2. **Clear descriptions**: Explain what you're adding and why
3. **Follow the format**: Match the existing style exactly
4. **Test your links**: Ensure all URLs work correctly
5. **Update the Table of Contents**: If adding a new category

## Code of Conduct

- Be respectful and constructive in discussions
- Focus on facts, not opinions about tools
- Avoid promotional language
- Help maintain the quality and objectivity of the list

## Questions?

If you have questions about contributing, feel free to open an issue with the "question" label.
